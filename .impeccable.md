## Design Context

### Users
Dual audience: **recruiters/employers** evaluating Kristian's fit for technical roles, and **developer peers** looking for a collaborator or interesting person to follow. Both groups arrive with curiosity and should leave with a clear sense of who Kristian is — not just what he's built. The goal is to make Kristian proud to hand out the URL.

### Brand Personality
**Bold. Human. Precise.**

- Confident without being corporate — this is a real person's space, not a LinkedIn profile
- Warm and approachable, with an engineer's attention to detail underneath
- Directional reference: Kristian's blog at kristian.pages.dev uses thematic coherence and restraint to create personality — this portfolio should channel that same spirit, dialled back to a professional register

### Aesthetic Direction
- **Light & airy** primary — generous whitespace, breathing room, high contrast type
- Not generic Bootstrap: the current Toha theme feels like a template; every design decision should feel intentional and personal
- Not flashy or gimmicky — no heavy parallax, no over-animated entrances, no visual noise
- Typography-led: strong font choices do more work than decoration
- Accent color palette is already good (`#248AAA` teal, with magenta/orange/green accents) — lean into these more boldly rather than the current muted off-whites
- Dark mode support should feel polished, not just inverted

### Design Principles

1. **Earned confidence** — Every element should signal craft. If it looks like a template, it fails. Prefer distinctive over safe.
2. **Whitespace is content** — Let things breathe. Density signals stress; space signals control.
3. **Typography carries the personality** — Use type scale and weight to create hierarchy and character. Don't rely on decoration.
4. **Warmth through specificity** — Personal details, real project context, and a human voice make this feel lived-in. Generic copy or stock patterns undermine the brand.
5. **Restraint over spectacle** — Animations and effects should be subtle and purposeful. If an effect doesn't improve comprehension or delight, remove it.

### Current Tech Stack Notes
- Raw HTML/CSS — no build step, no framework
- `index.html` — single-page site; `css/style.css` — all styles; `projects/<name>/index.html` — project pages
- Existing accent: `#248AAA` (teal/primary), `#1C2D41` (dark heading), `#3C4858` (body text)
